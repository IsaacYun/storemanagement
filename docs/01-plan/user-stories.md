# 사용자 스토리 (User Stories)

## 관리자 스토리

### 매장/근무자 관리

#### US-A01: 매장 등록
**As a** 관리자
**I want to** 새로운 매장을 등록하고 시급, 운영시간을 설정
**So that** 해당 매장의 근무자 관리를 시작할 수 있다

**Acceptance Criteria:**
- Given: 관리자가 로그인한 상태
- When: 매장 등록 폼에 이름, 시급, 운영시간, 만근보너스를 입력하고 저장
- Then: 새 매장이 생성되고 헤더 선택 목록에 표시됨

---

#### US-A02: 근무자 등록
**As a** 관리자
**I want to** 매장에 새 근무자를 등록
**So that** 해당 근무자의 스케줄과 급여를 관리할 수 있다

**Acceptance Criteria:**
- Given: 매장이 선택된 상태
- When: 근무자 정보(이름, 세금적용여부)를 입력하고 저장
- Then: 근무자가 해당 매장에 등록되고 스케줄 설정이 가능해짐

---

#### US-A03: 기본 스케줄 설정
**As a** 관리자
**I want to** 근무자의 주간 고정 스케줄을 설정
**So that** 매주 반복되는 근무 시간이 자동으로 적용된다

**Acceptance Criteria:**
- Given: 근무자 상세 페이지
- When: 요일별 시작/종료 시간을 설정
- Then: 해당 요일마다 스케줄이 달력에 자동 표시됨

---

### 스케줄 관리

#### US-A04: 달력으로 스케줄 확인
**As a** 관리자
**I want to** 월간 달력에서 모든 근무자의 스케줄을 한눈에 확인
**So that** 전체 근무 현황을 파악할 수 있다

**Acceptance Criteria:**
- Given: 매장이 선택된 상태에서 달력 페이지 접근
- When: 달력이 로드됨
- Then: 각 날짜에 해당 일의 근무자 목록이 표시됨

---

#### US-A05: 변동사항 입력 (미근무)
**As a** 관리자
**I want to** 근무자가 특정 날짜에 근무 불가할 때 미근무를 등록
**So that** 해당 날짜의 근무 시간이 차감된다

**Acceptance Criteria:**
- Given: 변동사항 입력 페이지
- When: 근무자, 날짜, 미근무 유형을 선택하고 저장
- Then: 해당 날짜에 미근무 뱃지가 표시되고, 월말 정산에 반영됨

---

#### US-A06: 대타 근무 등록
**As a** 관리자
**I want to** 대타 근무를 등록할 때 원래 근무자와 대타 근무자를 모두 기록
**So that** 양쪽 근무자의 급여가 정확히 계산된다

**Acceptance Criteria:**
- Given: 변동사항 입력 페이지에서 대타 유형 선택
- When: 원래 근무자, 대타 근무자, 근무 매장, 시간을 입력
- Then: 원래 근무자에게 미근무, 대타 근무자에게 대타근무가 각각 기록됨

---

#### US-A07: 타매장 대타 처리
**As a** 관리자
**I want to** 다른 매장에서 대타 근무한 경우 양쪽 매장에서 확인
**So that** 어느 매장에서든 해당 근무 내역을 볼 수 있다

**Acceptance Criteria:**
- Given: 금곡 소속 근무자가 두정동에서 대타 근무 등록
- When: 금곡 매장 선택
- Then: 해당 근무자의 대타 근무 내역이 표시됨 (두정동 대타 표시)
- When: 두정동 매장 선택
- Then: 해당 날짜에 대타 근무자가 표시됨

---

### 월말 정산

#### US-A08: 월별 정산 조회
**As a** 관리자
**I want to** 해당 월의 모든 근무자 급여를 한 테이블에서 확인
**So that** 전체 급여 현황을 파악하고 지급할 수 있다

**Acceptance Criteria:**
- Given: 정산 페이지에서 년/월 선택
- When: 정산 테이블 로드
- Then: 근무자별 총근무시간, 기본급, 추가지급, 세금, 실수령액이 표시됨

---

#### US-A09: 카카오톡 메시지 복사
**As a** 관리자
**I want to** 근무자별 급여 내역을 포맷된 텍스트로 복사
**So that** 카카오톡으로 바로 전송할 수 있다

**Acceptance Criteria:**
- Given: 근무자 정산 상세 페이지
- When: "카카오톡 복사" 버튼 클릭
- Then: 포맷된 급여 안내 텍스트가 클립보드에 복사됨

---

#### US-A10: 조기 정산
**As a** 관리자
**I want to** 월말 며칠 전에도 남은 예정 스케줄을 포함해 정산
**So that** 미리 급여를 계산하고 준비할 수 있다

**Acceptance Criteria:**
- Given: 월 중간 시점에서 정산 페이지 접근
- When: "예정 스케줄 포함" 옵션 활성화
- Then: 기록된 변동사항 + 남은 기본 스케줄을 합산한 급여 표시

---

### 관리 기능

#### US-A11: 활동 로그 조회
**As a** 관리자
**I want to** 관리자들의 수정 이력을 확인
**So that** 누가 언제 어떤 변경을 했는지 추적할 수 있다

**Acceptance Criteria:**
- Given: 관리 > 활동 로그 페이지
- When: 로그 목록 조회
- Then: 관리자별 접속, 수정 내역이 시간순으로 표시됨

---

## 근무자 스토리

#### US-W01: 내 스케줄 확인
**As a** 근무자
**I want to** 내 근무 스케줄을 달력으로 확인
**So that** 언제 근무하는지 알 수 있다

**Acceptance Criteria:**
- Given: 근무자로 로그인
- When: 스케줄 페이지 접근
- Then: 본인의 근무 스케줄만 표시됨

---

#### US-W02: 변동사항 직접 입력
**As a** 근무자
**I want to** 지각이나 조퇴 등 변동사항을 직접 입력
**So that** 관리자에게 따로 연락하지 않아도 된다

**Acceptance Criteria:**
- Given: 근무자로 로그인한 상태
- When: 변동사항 입력 페이지에서 본인의 변동사항 등록
- Then: 변동사항이 저장되고 관리자가 확인 가능

---

#### US-W03: 내 급여 확인
**As a** 근무자
**I want to** 내 월별 급여 내역을 확인
**So that** 받을 급여를 미리 알 수 있다

**Acceptance Criteria:**
- Given: 근무자로 로그인
- When: 정산 페이지 접근
- Then: 본인의 근무시간, 급여 내역만 표시됨
