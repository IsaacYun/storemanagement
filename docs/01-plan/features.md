# 기능 명세서 (Features)

## MoSCoW 우선순위

### MUST HAVE (P0) - 필수

#### F-001: 매장 관리
- 매장 CRUD (생성, 조회, 수정, 삭제)
- 필드: 이름, 시급, 만근보너스, 운영시간
- 매장 활성/비활성 상태 관리

#### F-002: 근무자 관리
- 근무자 CRUD
- 필드: 이름, 소속매장, 세금적용여부, 역할(admin/worker)
- 근무자 활성/비활성 상태 관리

#### F-003: 기본 스케줄 설정
- 요일별 근무 시간 설정 (시작/종료)
- 근무자당 하나의 주간 스케줄 패턴
- 스케줄 활성/비활성

#### F-004: 변동사항 입력
- 변동 유형: 미근무, 추가근무, 대타, 지각, 조퇴, 식대, 주휴수당
- 필드: 근무자, 날짜, 유형, 시간, 메모
- 대타: 원래 근무자 + 대타 근무자 + 근무 매장

#### F-005: 스케줄 달력 뷰
- 월간 달력 형태
- 일자별 근무자 목록 표시
- 변동사항 뱃지 표시 (미근무: 빨강, 대타: 파랑 등)
- 월 이동 (이전/다음)

#### F-006: 월말 정산
- 근무시간 자동 계산 (기본 + 변동)
- 급여 계산 (시급 x 시간 + 추가지급 - 세금)
- 근무자별 정산 테이블

#### F-007: 인증 시스템
- 이메일/비밀번호 로그인
- 역할 기반 접근 제어
- 로그인/로그아웃

#### F-008: RLS 보안
- 관리자: 전체 데이터 접근
- 근무자: 본인 데이터만 접근
- 변동사항: 근무자는 본인 것만 입력 가능

## 기능별 상세 명세

### F-004: 변동사항 입력 상세

#### 변동 유형 정의

| 유형 | 코드 | 설명 | 시간/금액 | 급여 영향 |
|------|------|------|----------|----------|
| 미근무 | `absence` | 근무 예정일 결근 | 분 (기본스케줄 시간) | 차감 |
| 추가근무 | `overtime` | 연장, 조기출근 | 분 | 추가 |
| 대타 | `substitute` | 다른 사람 대신 근무 | 분 | 추가 |
| 지각 | `late` | 늦게 출근 | 분 | 차감 |
| 조퇴 | `early_leave` | 일찍 퇴근 | 분 | 차감 |
| 식대 | `meal_allowance` | 복지 추가 지급 | 금액 | 추가 |
| 주휴수당 | `weekly_holiday_pay` | 평일 만근 수당 | 금액 | 추가 |

#### 대타 근무 처리 로직
1. 원래 근무자: `absence` 변동 자동 생성
2. 대타 근무자: `substitute` 변동 생성
3. `work_store_id`: 실제 근무한 매장 (타매장 대타 시)
4. 양쪽 매장에서 조회 가능

### F-006: 월말 정산 계산 로직

#### 근무시간 계산
```
총근무시간 = 기본스케줄
           - 미근무
           + 추가근무
           + 대타근무
           - 지각
           - 조퇴
```

#### 급여 계산
```
기본급 = 시급 × (총근무시간 / 60)
세전급여 = 기본급 + 식대 + 주휴수당 + 만근보너스
세금 = 세전급여 × 3.3% (적용 시)
실수령액 = 세전급여 - 세금
```

```
  ─────────────

▶ 실수령액: 916,232원
━━━━━━━━━━━━━━━
```
