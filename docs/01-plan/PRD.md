# 매장 근무 관리 시스템 - 제품 요구사항 정의서 (PRD)

## 1. 개요

### 1.1 배경
- 3개 매장 운영, 매장별 4-5명 근무자 관리
- 현재 수기로 근무 변동사항 메모, 월말 근무시간/급여 수동 계산
- 문제점: 시간 소요 (월 4시간+), 계산 실수 빈발

### 1.2 목표
- 근무 스케줄 및 변동사항 디지털 관리
- 월말 급여 정산 자동화
- 근무자에게 급여 내역 카카오톡으로 간편 전송

### 1.3 성공 지표
- 월말 정산 시간: 4시간 → 30분 이내
- 계산 오류: 0건
- 모바일 사용성: 100%

---

## 2. 사용자 정의

### 2.1 관리자 (Admin)
- 매장 운영자 (본인 포함 여러 명)
- 모든 매장에 대한 전체 권한
- 주요 업무: 스케줄 설정, 변동사항 입력, 월말 정산

### 2.2 근무자 (Worker)
- 각 매장의 아르바이트생
- 본인 스케줄 조회, 본인 변동사항 입력 가능
- 급여 내역 확인

---

## 3. 기능 요구사항

### 3.1 매장 관리
| 기능 | 설명 |
|------|------|
| 매장 등록 | 이름, 운영시간, 시급, 만근보너스 설정 |
| 매장 선택 | 헤더에서 매장 선택 후 해당 매장 데이터만 표시 |
| 매장 수정 | 시급, 보너스 등 설정 변경 |

### 3.2 근무자 관리
| 기능 | 설명 |
|------|------|
| 근무자 등록 | 이름, 소속 매장, 세금 적용 여부 |
| 스케줄 설정 | 요일별 근무 시간 (고정 스케줄) |
| 계정 연동 | 근무자 로그인 계정 생성 (선택) |

### 3.3 스케줄 조회
| 기능 | 설명 |
|------|------|
| 달력 뷰 | 월간 달력에 일자별 근무자 표시 |
| 리스트 뷰 | 날짜별 근무 목록 |
| 변동사항 표시 | 뱃지로 미근무/대타/추가근무 구분 |

### 3.4 변동사항 입력
| 변동 유형 | 설명 |
|----------|------|
| 미근무 | 근무 예정일에 근무 불가 |
| 추가근무 | 연장 근무, 조기 출근 |
| 대타근무 | 다른 근무자 대신 근무 (타매장 포함) |
| 지각 | 예정 시간보다 늦게 출근 |
| 조퇴 | 예정 시간보다 일찍 퇴근 |
| 식대 | 복지 형태의 추가 지급 |
| 주휴수당 | 평일 5일 만근 시 수동 입력 |

### 3.5 월말 정산
| 기능 | 설명 |
|------|------|
| 근무시간 계산 | 기본스케줄 + 변동사항 반영 |
| 급여 계산 | 시급 x 시간 + 추가지급 - 세금 |
| 통계 테이블 | 근무자별 요약 표시 |
| 카카오톡 복사 | 포맷된 텍스트 클립보드 복사 |
| 조기 결산 | 월말 전 예정 스케줄 포함 정산 |

### 3.6 활동 로그
| 기능 | 설명 |
|------|------|
| 로그 기록 | 관리자 접속, 수정 이력 자동 저장 |
| 로그 조회 | 관리자 전용 로그 목록 조회 |

### 4.1 성능
